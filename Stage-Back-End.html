Back-End & API

Le back-end de l'application repose sur Django REST Framework, un framework robuste pour créer des API REST en Python. J’ai conçu les modèles de données (models) pour représenter les entités clés du projet, puis utilisé les sérialiseurs pour exposer ces objets sous forme de JSON.
Chaque entité comme les manifestations, autorisations et demandes de matériel a été traitée via un ModelViewSet dédié.



Pour éviter les doublons entre les demandes d’autorisations et celles de matériel, j’ai mis en place un système d’identifiants préfixés. Cette stratégie a facilité la gestion des suppressions et des requêtes conditionnelles côté front-end.
Les routes API sont déclarées dans un fichier centralisé urls.py et testées avec Postman pour garantir leur fiabilité. J’ai également utilisé des filtres dynamiques pour limiter certaines requêtes (par ex. éviter les doublons de demandes sur une même manifestation).



J’ai adopté une structure modulaire pour les modèles, en les répartissant dans plusieurs fichiers et en centralisant leur import dans __init__.py. Cela rend le code plus lisible et maintenable.
Grâce à cette approche, les nouvelles entités ou évolutions de schéma peuvent être intégrées rapidement, tout en respectant la structure imposée par le projet MPT.




Outils & pratiques

API : Django REST Framework
Outils de test : Postman
Base locale : PostgreSQL via DataGrip
Méthodologie : Agile (réunions toutes les 2 semaines)
Versionnage : GitLab (Git Flow avec branches par développeur)
